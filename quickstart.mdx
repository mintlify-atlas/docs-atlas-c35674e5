# Quick Start Guide

This guide will walk you through customizing the template and building your first feature. By the end, you'll have a working understanding of the template's structure and capabilities.

<Note>
  Make sure you've completed the [Installation](/getting-started/installation) before proceeding.
</Note>

## Start the Development Server

<Steps>
  <Step title="Run the dev server">
    Start your Next.js development server:

    <CodeGroup>
    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>

    Your application will be running at [http://localhost:3000](http://localhost:3000)
  </Step>

  <Step title="View the landing page">
    Open your browser and navigate to `http://localhost:3000`. You should see the default landing page with:

    - A hero section with gradient text
    - Feature cards showcasing template capabilities
    - Installation instructions
    - A footer with links

    <Tip>
      The page auto-updates as you edit files, thanks to Next.js Fast Refresh!
    </Tip>
  </Step>
</Steps>

  <Step title="Update the hero text">

    ```tsx src/app/(website)/(landing)/page.tsx

    ```

    Change the text to match your project. The gradient spans use Tailwind's background clip utilities.
  </Step>

  <Step title="Customize the description">
    Update the description text below the heading:

    ```tsx src/app/(website)/(landing)/page.tsx

    </p>
    ```

  <Step title="Modify feature cards">
    The features array (lines 23-65) defines the feature cards. Customize them:

    ```tsx src/app/(website)/(landing)/page.tsx
    const features = [
      {
        icon: Palette,
        title: "Your Feature",
        description: "Description of your amazing feature",
      },
      // Add more features...

    ```

    Available icons from `lucide-react`: `Palette`, `Sparkles`, `Database`, `Shield`, `Code`, `Zap`, `Globe`, `Rocket`, and many more.
  </Step>
</Steps>

## Make Your First API Request

The template includes a custom Axios API client. Let's use it to fetch data.

<Steps>
  <Step title="Create a new component">
    Create a new component file:

    ```tsx src/components/example-api.tsx
    "use client";

    import { useState } from "react";
    import api from "@/lib/API/api";
    import { Button } from "@/components/ui/button";

    export function ExampleApi() {
      const [data, setData] = useState<any>(null);
      const [loading, setLoading] = useState(false);

      const fetchData = async () => {
        setLoading(true);
        const response = await api.get("/api/example");
        
        if (response.status === 200) {
          setData(response.data);
        }
        setLoading(false);
      };

      return (
        <div className="space-y-4">
          <Button onClick={fetchData} disabled={loading}>
            {loading ? "Loading..." : "Fetch Data"}
          </Button>
          
          {data && (
            <pre className="bg-muted p-4 rounded-lg">
              {JSON.stringify(data, null, 2)}
            </pre>
          )}
        </div>
      );
    }
    ```
  </Step>

  <Step title="Add to your page">
    Import and use the component in your landing page:

    ```tsx src/app/(website)/(landing)/page.tsx
    import { ExampleApi } from "@/components/example-api";

    export default function IndexPage() {
      // ... existing code

      return (
        <div className="bg-background min-h-screen px-4">
          {/* ... existing content */}
          
          <ExampleApi />
        </div>
      );
    }
    ```
  </Step>

  <Step title="Try the API">
    Click the "Fetch Data" button on your page. The template includes an example API route at `/api/example` that returns:

    ```json
    {
      "message": "Hello World"
    }
    ```

    <Check>
      The API client automatically handles authentication headers and error responses!
    </Check>
  </Step>
</Steps>

## Create an API Route with Validation

Let's create a new API route with Zod validation and rate limiting.

<Steps>
  <Step title="Create the route file">
    Create a new API route:

    ```typescript src/app/api/contact/route.ts
    import { NextRequest } from "next/server";
    import {
      successResponse,
      handleAndReturnErrorResponse,
      validateRequestBody,
    } from "@/lib/API/handler";
    import { withRateLimit } from "@/lib/API/middleware";
    import { contactFormSchema } from "@/lib/validation/schemas";

    export const POST = withRateLimit(async (req: NextRequest) => {
      try {
        // Validate the request body against the schema
        const validatedData = await validateRequestBody(contactFormSchema, req);

        // Process the contact form (send email, save to DB, etc.)
        console.log("Contact form data:", validatedData);

        return successResponse({
          message: "Contact form submitted successfully",
          status: 200,
          data: validatedData,
        });
      } catch (error) {
        return handleAndReturnErrorResponse(error);
      }
    });
    ```

    <Note>
      The `withRateLimit` middleware automatically applies rate limiting. The `validateRequestBody` function ensures type-safe data validation.
    </Note>
  </Step>

  <Step title="Understand the schema">
    The `contactFormSchema` is defined in `src/lib/validation/schemas.ts`:

    ```typescript src/lib/validation/schemas.ts
    export const contactFormSchema = z.object({
      name: z.string().min(2, "Name must be at least 2 characters"),
      email: z.email("Invalid email address"),
      message: z.string().min(10, "Message must be at least 10 characters"),
    });
    ```

    This provides automatic validation with helpful error messages.
  </Step>

  <Step title="Call from the frontend">
    Use the API client to call your new endpoint:

    ```tsx
    import api from "@/lib/API/api";

    const submitContactForm = async (formData: ContactFormInput) => {
      const response = await api.post("/api/contact", formData);
      
      if (response.status === 200) {
        console.log("Success!", response.data);
      } else {
        console.error("Error:", response.error);
      }
    };
    ```
  </Step>
</Steps>

## Understanding the Project Structure

```bash
src/
├── app/
│   ├── (website)/          # Website routes group
│   │   ├── (landing)/      # Landing page
│   │   ├── 404/            # Custom 404 page
│   │   └── error.tsx       # Error boundary
│   ├── api/                # API routes
│   │   └── example/        # Example API endpoint
│   ├── layout.tsx          # Root layout
│   └── not-found.tsx       # 404 handler
├── components/
│   └── ui/                 # 30+ ShadCN UI components
├── lib/
│   ├── API/                # API client & utilities
│   │   ├── api.ts          # Axios wrapper
│   │   ├── handler.ts      # Response handlers
│   │   ├── middleware.ts   # Auth & rate limit middleware
│   │   └── rate-limit.ts   # Rate limit configuration
│   ├── auth/               # Authentication utilities
│   ├── cache/              # Redis client
│   ├── database/           # Prisma client
│   ├── validation/         # Zod schemas
│   └── custom/             # Custom providers & hooks
├── constants/              # Site configuration
prisma/
└── schema.prisma           # Database schema
```

## Key Concepts

### API Client

The custom Axios client (`src/lib/API/api.ts`) provides:
- Automatic authentication headers
- Type-safe responses
- Error handling
- Base URL configuration

```typescript
import api from "@/lib/API/api";

// GET request
const response = await api.get<DataType>("/api/endpoint");

// POST request
const response = await api.post<ResponseType, BodyType>(
  "/api/endpoint",
  { data: "value" }
);
```

### Validation

Zod schemas in `src/lib/validation/schemas.ts` provide type-safe validation:

```typescript
import { z } from "zod";

export const mySchema = z.object({
  email: z.email("Invalid email"),
  name: z.string().min(2),
});

export type MyInput = z.infer<typeof mySchema>;
```

### Rate Limiting

API routes are protected with rate limiting:

```typescript
import { withRateLimit } from "@/lib/API/middleware";

export const POST = withRateLimit(async (req: NextRequest) => {
  // Your handler code
});
```

Rate limits:
- **Default**: 100 requests per 15 minutes
- **Strict**: 5 requests per minute
- Supports both Redis and in-memory storage

### Smooth Scroll

Lenis smooth scroll is enabled by default on desktop (>1024px). Configure it in your constants:

```typescript src/constants/index.ts
export const website = {
  enableLenis: true, // Enable/disable smooth scroll
  accentColor: "#B16CEA",
  // ... other config
};
```
